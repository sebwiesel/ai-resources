# Skills

This directory is intentionally starting empty.

Skills in this repository are meant for professional reuse, so new additions should pass review before they are merged. Once approved, each skill should live in its own folder and include a `SKILL.md` file as the canonical definition.

Recommended future structure:

```text
skills/
└── example-skill/
    ├── SKILL.md
    ├── examples/
    └── notes.md
```

Before proposing a new skill, review [CONTRIBUTING.md](../CONTRIBUTING.md), especially the sections on scope, validation, and failure modes.
